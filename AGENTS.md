# AGENTS.md

本文件是本仓库的 Codex 项目级工作指令。之后在本仓库中让 Codex 修改代码、分析日志或编写脚本时，应默认遵守这些约定。

## 语言

- 默认使用简体中文与用户交流。
- 代码中的变量名、函数名、类名、文件名、命令行参数、配置键、日志字段保持英文。
- 新增或修改的注释、docstring、脚本说明优先使用简体中文。

## 注释风格

- 关键函数都需要写注释，阐明功能、用法、实现原理。
- 不写逐行解释式注释，但也不要一个脚本文件中就没几个注释。
- 仅在复杂逻辑、实验参数、张量 shape、数据流、边界条件、随机性控制、非显然实现处添加中文注释。
- 修改已有英文注释时，如果该注释与当前改动相关，可顺手改为简体中文；无关注释不要为了统一语言而大面积重写。
- 注释应说明“为什么这样做”或“这里需要注意什么”，避免重复代码本身已经表达清楚的内容。

## 代码修改原则

- 优先遵循仓库现有代码风格、目录结构和命名习惯。
- 修改范围应尽量聚焦用户请求，不做无关重构。
- 不随意删除、覆盖实验日志、模型权重、结果文件或中间产物。
- 涉及训练、评估、数据增强、净化流程时，应尽量保持实验可复现性，明确随机种子、参数来源和输出路径。
- 修改 shell 脚本时，应注意变量引用、路径空格、失败退出行为和日志文件名是否稳定。

## 验证

- 修改代码后，优先运行与改动直接相关的最小验证命令。
- 验证时，可以使用 conda 中的 torch 环境进行验证。
- 如果完整训练或评估成本较高，应给出可快速检查语法、参数解析或 dry-run 行为的命令。
- 如果验证命令无法运行，应说明失败原因和仍然存在的风险。

## 汇报

- 完成修改后，简要说明改了哪些文件、改动目的和验证结果。
- 若发现用户工作区已有未提交改动，不要回退或覆盖；只在必要时说明与本次任务相关的影响。
