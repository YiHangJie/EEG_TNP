# EEG_TNP / PTR 动态 rank 研究逻辑与进展梳理

本文档面向论文讨论和组会交流，目标不是复述所有实验细节，而是把当前项目的研究逻辑、证据链、阶段性结论和后续方向串起来。

## 一句话定位

本项目研究的是 EEG 对抗防御中的输入级测试时净化：用 Progressive Tensor Ring（PTR）对 EEG 样本做低秩张量重构，在送入分类器前削弱对抗扰动；进一步研究如何为不同样本自适应选择净化 rank，从而缓解 fixed rank 带来的 clean accuracy 与 robust accuracy 权衡。

> PTR 低秩净化有效，但 rank 决定净化强度；因此需要一个可解释、可复现、可在线化的 sample-wise rank selection 规则。

## 当前阶段定位：方法冻结，转向证据补强

截至 2026-07-04，项目主方法已经基本确定。后续默认不再继续发明新的主方法或大幅改动核心 pipeline，而是围绕已经确定的方法链路补齐论文需要的证据：

- 补完整主实验：跨 seed、fold、backbone、dataset、epsilon 和攻击设置。
- 跑必要消融：sensitive layer selection、rank schedule、online AT、训练 rank 分布、净化 rank、baseline 公平性等。
- 做超参数与稳定性检查：只在明确范围内搜索，并记录随机种子、搜索空间和输出路径。
- 完善可视化：主表、误差棒、rank/seed/backbone 对比、净化前后分布和训练曲线。
- 做低秩与信号分析：rank、频谱、trial-level 低秩结构、扰动和净化残差的可解释分析。
- 提升多 GPU 实验效率：优先使用可续跑、可 shard、日志实时写入的 pipeline，并让独立 seed/backbone/rank/baseline 任务并行运行。

对后续 AI 协作的约束是：除非用户明确要求或发现影响结论的 bug/公平性问题，否则不要默认提出新的主方法结构。更合适的工作方式是帮助补实验、调度多 GPU、整理结果、做可视化、补分析和维护文档闭环。

## 背景与核心矛盾

EEG 对抗防御常见方法仍以对抗训练为主，例如 Madry、TRADES、ABAT 等。这类方法直接优化分类器参数，能够提升某些攻击设置下的鲁棒性，但存在几个限制：

- 防御往往绑定训练阶段的攻击方式、扰动强度和分类器结构。
- EEG 信号低信噪比、非平稳、跨被试差异强，分类边界本身比较脆弱。
- 对抗训练主要让分类器适应扰动，并不直接修复输入信号。
- clean accuracy 与 robust accuracy 很容易出现 trade-off。

本项目转向输入级净化：先用 PTR 把输入 EEG 投影到较低秩的张量结构，再分类。直觉是 EEG 中任务相关成分具有一定时空结构，而对抗扰动更容易表现为不稳定、非结构化或高频细节；低秩重构可以保留主导结构，同时抑制一部分扰动。

但 PTR 净化有一个核心矛盾：

- rank 太低：净化强，可能抑制扰动，但也会损伤 clean 语义。
- rank 太高：保真好，clean acc 往往更高，但可能恢复扰动或对分类 margin 不利。

因此，fixed rank 很难同时最优，动态 rank 是当前项目的主要研究问题。

## 方法链路

当前方法可以分成三层。

### 1. 输入级 PTR 净化

给定 EEG 样本，PTR 用张量环低秩结构做测试时重构。净化后的样本再输入下游 EEGNet 分类器。

基本作用：

- 降低输入中的非主导结构。
- 用 rank 控制重构容量。
- 通过测试时优化避免重新训练净化器。

### 2. 净化样本感知的分类器训练

仓库中当前主要 checkpoint 使用了对抗训练和净化样本一致性思想。训练分类器时不仅考虑 clean/adv 样本，也让模型适应 PTR 净化后的输入分布；同时通过 logits consistency 减少净化引起的预测漂移。

这一步的作用是降低“净化样本分布”和“原始训练分布”之间的不匹配。

### 3. 动态 rank selection

这是近期工作的核心。PTR 的 rank 决定净化强度，因此需要从固定 rank 走向 sample-wise rank。

目前已经测试过三类思路：

- hard rank growth：从低 rank 到高 rank 逐步净化，并用外部规则选 rank。
- 诊断指标：JS、MSE、margin、entropy、高频增量等。
- soft rank mask：把离散 active rank 放松成可微 effective rank。

## 关键实验设置

当前主要结论来自：

- dataset：`thubenchmark`
- model：`eegnet`
- split：`fold0`
- seed：`42`
- attack：`autoattack`
- 主要扰动强度：`eps=0.03`
- 扩展复验：`eps=0.1`
- 样本量：多数核心 rank-selection 实验为 `sample_num=512`
- 主要 checkpoint：`consistancy_rank25-30_n512_eps0p03`

这些结果足以支撑当前方法选择，但还不能替代跨 seed/fold 的最终论文验证。

## 阶段性证据链

### 阶段 1：固定 rank 证明 rank trade-off 真实存在

`EXP-001/009` 显示 fixed rank 下 clean/robust/MSE 的最优点并不一致。

在 `EXP-001` 中，净化前基线为：

| setting | accuracy |
| --- | ---: |
| clean before purification | `0.943359` |
| adv before purification | `0.773438` |

固定 rank 的代表结果：

| rank | adv acc | clean acc | Adv MSE |
| ---: | ---: | ---: | ---: |
| 15 | `0.820313` | `0.873047` | `0.132929` |
| 20 | `0.835938` | `0.898438` | `0.086367` |
| 25 | `0.845703` | `0.906250` | `0.058501` |
| 30 | `0.833984` | `0.927734` | `0.040879` |
| 35 | `0.835938` | `0.931641` | `0.028913` |
| 40 | `0.832031` | `0.933594` | `0.020669` |

关键观察：

- robust acc 在 `rank25` 附近最好。
- clean acc 和 MSE 更偏高 rank。
- rank 继续升高，MSE 继续下降，但 robust acc 不一定提高。

`EXP-009` 的 full-sweep n512 复现了同类趋势：`rank20` 的 adv acc 最高为 `0.837891`，而 clean acc 在 `rank35` 达到 `0.916016`。

这说明 rank 不是单纯越高越好；它确实是净化强度和语义保真之间的调节变量。

### 阶段 2：hard rank growth 证明动态 rank 有机会超过固定 rank

`IDEA-001 / EXP-001` 将 PTR 改造为 rank growth，从低 rank 逐步增加 rank，并用分类稳定性 JS 规则早停。

代表结果：

| method | adv acc | clean acc | mean rank |
| --- | ---: | ---: | ---: |
| fixed `rank25` | `0.845703` | `0.906250` | `25` |
| dynamic `js=0.02` | `0.849609` | `0.910156` | `12.104` |
| dynamic `js=0.005` | `0.851563` | `0.914063` | `15.972` |

结论：

- 动态 rank 的核心假设得到部分支持。
- `js=0.005` 用接近 `rank20` 的平均 rank 达到高于 fixed `rank25` 的 robust acc。
- 但 early stop 的 MSE 较高，说明纯分类稳定性更偏 robust，不够保真。

### 阶段 3：MSE gate 改善保真，但提高计算成本

`EXP-002` 修复 MSE gate 后，`js=0.02,mse=0.08` 得到：

| method | adv acc | clean acc | Adv MSE | mean rank |
| --- | ---: | ---: | ---: | ---: |
| `js=0.02,mse=0.08` | `0.851563` | `0.916016` | `0.064028` | `27.344` |

相对纯 JS：

- robust acc 达到 `0.851563`，与 `js=0.005` 持平。
- clean acc 提升到 `0.916016`。
- MSE 大幅下降。
- 代价是 mean rank 从约 `12~16` 升到 `27.344`，计算成本明显上升。

这说明 MSE 是有效的保真约束，但不能单独解决低成本动态选择问题。

### 阶段 4：高频增量是有用诊断，但不是 adversarial-specific 停止信号

`IDEA-002 / EXP-005/006/009` 测试了 rank 增长过程中新增恢复成分的高频能量占比。

发现：

- 高频占比随 rank 增大整体上升，例如 `5->10` 约 `0.083`，`35->40` 约 `0.224`。
- 但 clean 与 adv 曲线几乎重合。
- paired `adv-clean` delta 接近 `0`。
- bootstrap 和 margin 联合分析也没有支持“某个高频段是 adversarial-specific 扰动恢复”的说法。

结论：

- 高频指标可以作为 rank-growth 过程的频谱诊断。
- 不能单独作为停止准则。
- 不能把“高频上升”直接讲成“开始恢复对抗扰动”的强证据。

### 阶段 5：Optuna 离线 replay 找到更强的 hard selector 候选

`IDEA-003 / EXP-007~010` 对 JS、MSE、margin 及其组合做离线 rank-selection 调参。

`EXP-010` 在 n512 上得到当前最重要结论：

| selector | all adv acc | all clean acc | all MSE | mean rank |
| --- | ---: | ---: | ---: | ---: |
| fixed `rank20` | `0.837891` | `0.865234` | `0.108970` | `20` |
| fixed `rank35` | `0.835938` | `0.916016` | `0.039158` | `35` |
| `threshold` | `0.847656` | `0.884766` | `0.113463` | `19.434` |
| `js_mse` | `0.845703` | `0.890625` | `0.129503` | `17.671` |
| `mse_only` | `0.839844` | `0.876953` | `0.090733` | `22.349` |

关键判断：

- `threshold` 全量 adv acc 最高，为 `0.847656`。
- `js_mse` adv acc 很接近，为 `0.845703`，但 mean rank 更低。
- `mse_only` MSE 更低，但 robust acc 不如 `threshold/js_mse`。
- `margin_only` 往往退到高 rank，更像保真对照，不适合作为低成本 adaptive selector。
- `EXP-007` 的 full score 在 n512 上迁移失败，不应作为默认主线。

因此当前 hard rank-growth 主候选是 `threshold` 和 `js_mse`。

### 阶段 6：entropy 可诊断，不适合作为主 rank-selection 信号

`IDEA-004 / EXP-011` 测试预测熵。

结果显示：

- `js_mse_entropy` 和 `score_entropy` 没有超过 `threshold`。
- `entropy_only` 和 `entropy_margin` 经常选择高 rank，类似高保真/低熵对照。
- entropy 有诊断价值，但不应作为主选择信号。

结论：保留 entropy 字段和分析脚本，但不把它写入默认在线 rank-selection。

### 阶段 7：eps=0.1 复验显示 selector 不应直接过拟合 eps=0.03

`EXP-012/013` 将 pipeline 扩展到更强攻击 `eps=0.1`。

关键结果：

- `eps=0.1` 下，`threshold/js_mse` 不再复现 `eps=0.03` 上的明显优势。
- `score`、`js_mse`、`js_mse_entropy`、fixed `rank15` 的 robust acc 差距很小，约在 `0.796875~0.798828`。
- 高 rank 会提升 clean acc、降低 MSE/entropy，但 robust acc 随 rank 增大下降更明显。
- 在线 `js_mse` 早停能减少计算：真实在线早停墙钟时间约 `1:09:11`，相比 full-sweep 约 `2:02:32` 明显更短；但 adv acc 为 `0.78125`，低于离线 replay 的最优区间。

结论：

- 在线早停可以作为省计算候选。
- 不能把单个 eps 上的 selector 直接作为默认策略。
- 后续必须区分 accuracy-oriented selection 和 compute-oriented early stopping。

### 阶段 8：soft-rank mask 工程可行，但当前目标不够强

`IDEA-005 / EXP-014` 实现了 `PTR_3d_rank_soft_mask`：

- 学习连续 `rho`。
- 用 `g_i = sigmoid((rho - i) / temperature)` 形成 soft-prefix mask。
- 目标只用 `MSE + rank_cost`。

结果：

| rank penalty | clean acc | adv acc | all MSE | mean effective rank |
| --- | ---: | ---: | ---: | ---: |
| `0.0` | `0.900391` | `0.832031` | `0.058451` | `19.536` |
| `0.001` | `0.886719` | `0.830078` | `0.059238` | `19.330` |
| `0.003` | `0.894531` | `0.828125` | `0.060793` | `18.939` |
| `0.01` | `0.896484` | `0.837891` | `0.065855` | `17.715` |

结论：

- soft gate 工程路径成立，rank penalty 能压低 effective rank。
- 但 effective rank 分布很窄，clean/adv 均值几乎重合。
- rank 与 MSE 强相关，但与分类正确性相关很弱。
- best soft-mask adv acc `0.837891` 低于 hard selector 的 `threshold=0.847656` 和 `js_mse=0.845703`。

因此，`MSE + rank cost` 的 soft mask 不适合作为当前 accuracy-oriented 主线。它可以作为可微 rank gate 原型保留，但若继续推进，需要换更有结构的目标，而不是只 sweep penalty。

## 当前总体结论

1. PTR 低秩净化对 EEG 对抗防御是有效方向，但 rank 是核心控制变量。

2. fixed rank 存在不可忽视的 trade-off：低/中 rank 更利于 robust，高 rank 更利于 clean 和 MSE。

3. hard rank growth 证明了 sample-wise rank selection 的价值：在 `eps=0.03` n512 上，`threshold/js_mse` 能超过固定 rank robust acc，并保持较低 mean rank。

4. 单纯保真指标不够。MSE 可以改善 clean/MSE，但单独优化 MSE 或 rank cost 不能保证 robust acc。

5. 高频和 entropy 都有诊断价值，但目前都不适合作为主 rank-selection 信号。

6. soft-mask 是一个优雅的工程架构，但当前 `MSE + rank cost` 目标没有产生足够强的样本级动态 rank 分配，不宜作为论文主线。

7. 当前最有论文价值的方向是：围绕 hard rank-growth，提出可解释、可在线化、可跨设置复验的动态 rank selection 机制。

## 可以对外讨论的贡献点

### 贡献 1：把 EEG 对抗防御从分类器中心推进到输入级低秩净化

PTR 净化直接作用于输入信号，可以和已有对抗训练分类器组合，减少完全依赖分类器参数鲁棒化的问题。

### 贡献 2：证明 rank 是 EEG 张量净化中的关键防御控制变量

实验显示 robust 最优 rank 与 clean/MSE 最优 rank 不一致。这为动态 rank selection 提供了必要性，而不是单纯工程调参。

### 贡献 3：提出并验证 progressive rank-growth 的动态净化框架

rank-growth 不一次性固定 rank，而是在净化过程中观察分类稳定性和重构保真性，为不同样本选择净化强度。

### 贡献 4：系统排除多个直觉上合理但证据不足的信号

高频增量、entropy、soft MSE-rank gate 都经过实验验证，但目前不足以作为主策略。这些负结果能帮助论文叙事更严谨：不是堆指标，而是在寻找真正能解释 robust/clean trade-off 的选择信号。

### 贡献 5：形成 accuracy-oriented 与 compute-oriented 的区分

离线 selector 能找更优 rank，但计算成本高；在线早停能省计算，但可能牺牲一部分 robust acc。后续方法设计应显式区分这两个目标。

## 当前风险与局限

- 主要结果仍集中在 `thubenchmark fold0 seed42`，需要跨 seed/fold。
- `eps=0.03` 与 `eps=0.1` 下 selector 表现不同，说明规则可能对攻击强度敏感。
- 当前 best selector 仍是离线 replay 结果，在线实现与离线选择并不完全等价。
- clean 与 adv 的 rank 分布经常接近，说明现有规则更多是在处理“重构/分类稳定性”，还没有强烈区分 adversarial-specific 难度。
- 如果论文主张是“动态 rank 发现扰动结构”，当前证据还不够；更稳妥的主张是“动态 rank 通过稳定性与保真约束改善净化强度选择”。

## 建议下一步

当前下一步不再以“提出新方法”为目标，而是以“把既有方法的证据链补完整”为目标。实验组织上应优先利用多 GPU 环境，把可独立运行的 seed、backbone、rank shard、baseline 和分析任务拆开并行；所有长实验仍必须保证 `nohup`、实时日志、稳定输出目录和断点续跑。

### 短期：补齐主实验与必要消融

- 完成 `EXP-023/024` 这类 adaptive attack、backbone 和 baseline 补全实验。
- 围绕 RPCF_AT 固定主配置，补 sensitive layer selection、rank schedule、online AT、训练 rank 分布和净化 rank 的必要消融。
- 对已有超参数只做小范围、可记录、可复现的搜索，不把调参结果包装成新方法。

### 历史 rank-selector 分支：保留证据，不再作为新增主线

- `threshold`、`js_mse`、entropy、高频和 soft-rank 相关结果保留为方法探索证据与消融背景。
- 除非论文审稿、复现实验或用户明确要求，不再默认继续扩展新的 rank selector。
- 如需补充该分支，只做结果复核、可视化、运行时间统计或小范围超参数确认，不把它包装成新的主方法。

### 中期：做跨设置复验

- 至少覆盖更多 seed/fold。
- 同时覆盖 `eps=0.03` 和 `eps=0.1`。
- 检查 `threshold/js_mse/score` 在不同攻击强度下是否需要不同策略。

### 长期：如果继续 soft-rank，需要换目标

soft mask 本身可以保留，但不要只做 `MSE + lambda * rank`。更有价值的方向包括：

- rate-distortion knee：用边际重构收益判断最小充分 rank。
- 分类稳定性约束：让 soft gate 关注分类边界，而不只是 MSE。
- compute-aware objective：显式优化 accuracy 与净化时间/有效 rank 的 Pareto trade-off。

## 讨论时的推荐讲法

可以按下面逻辑讲：

1. EEG 对抗防御不能只靠分类器训练，输入净化有独立价值。
2. PTR 低秩净化有效，但 rank 决定净化强度。
3. fixed rank 的实验证明 robust 与 clean/MSE 的最优 rank 不一致。
4. 所以我们做 progressive rank growth，用分类稳定性和保真性做 sample-wise rank selection。
5. 在 `eps=0.03 n512` 上，`threshold/js_mse` 能超过固定 rank 的 robust acc，说明动态 rank 有价值。
6. 但高频、entropy、soft MSE-rank gate 目前都不是主信号，这些实验帮助我们收敛到更可解释的 hard selector。
7. 当前下一步是在线化 selector，并做跨 seed/fold/eps 复验，目标是形成一个可部署、可解释、计算成本可控的 EEG PTR 净化策略。

## 新阶段：从 rank selection 扩展到分类器的 purification-aware adaptation

`IDEA-007 / EXP-017` 引入 RPCF。它不再只研究“输入应该选择哪个 rank”，而是进一步研究：

> AT 分类器中的哪些层最难适应 EEG_TNP 引起的多 rank 分布变化，以及是否只更新这些层就足以提升净化后分类性能。

RPCF 的逻辑链为：

1. 标准 AT 提供基础 robust decision boundary。
2. 对同一训练样本及其一次生成的 adversarial sample 做多 rank 净化。
3. 用 clean feature 作为统一 anchor，衡量每个逻辑层面对 clean-purified 和 adversarial-purified views 的 feature shift。
4. 只更新 Top 40% purification-sensitive layers，避免全模型 fine-tuning 破坏已有 AT 表示。
5. 训练前期偏向 rank15 hard purification，后期逐渐偏向 rank40 mild purification，使模型先适应强分布偏移，再恢复对保真视图的拟合。

该方向与已有 rank-growth 主线互补：

- rank-growth 调整净化器对每个输入采用的表达能力。
- RPCF 调整分类器中负责吸收 purification-induced distribution shift 的参数子集。

首轮实验只比较 RPCF 与其 AT 初始化模型。若 RPCF 有效，下一步需要加入 full-layer fine-tuning、static rank weights 和跨 seed/fold 对照，区分收益究竟来自层选择、multi-rank data，还是 curriculum。

### RPCF 与 consistancy 的随机协议对齐

2026-06-12 起，RPCF 的训练和测试子集统一使用既有 consistancy 规则：
`selection_seed = seed + fold * 1000`。fine-tuning 的 batch shuffle 也只依赖
`seed_everything(seed)` 初始化的全局 RNG，不再使用 RPCF 私有 offset。此前
`EXP-017` 使用的测试净化 seed 为 `7049`，属于 legacy 结果；后续与 consistancy
做严格比较时必须使用修正后的 seed-aligned RPCF run。

### EXP-018 的公平重跑结论

在 `thubenchmark / EEGNet / fold0 / seed42 / eps0.03` 下，三条方法使用同一新训练
AT、各自 white-box AutoAttack 和同一 seed42 n512 净化子集完成重跑。RPCF 的最佳
checkpoint 停留在 epoch0，因此没有改变 AT 模型；其所有指标与普通 AT 完全一致。

consistancy 的未净化 AutoAttack accuracy 低于 AT，但在 rank25/30 净化后分别达到
`83.20%/83.40%`，高于 AT/RPCF 的 `81.84%/81.64%`。当前证据说明 consistancy
更擅长适配 purification-induced distribution shift，而现有 RPCF 的敏感层选择和
动态 rank 权重尚未转化为有效性能增益。

后续无早停 RPCF 复验改变了这一结论。相邻层 sensitivity 加 clean-teacher KL 固定
训练 100 epochs 后，rank25/30 purified adversarial accuracy 达到
`83.59%/84.18%`，超过 consistancy；rank30 purified clean accuracy 也达到
`93.75%`。但模型自身 AutoAttack accuracy 降至 `72.26%`，低于 AT 的 `77.86%`。
因此 RPCF 的机制能够增强 purification adaptation，但当前训练强度会损伤原始 AT
decision boundary。下一阶段的核心问题是控制这两个目标的 Pareto trade-off。

进一步去除 sensitive-layer selection、改为微调 100% 参数后，rank25 purified
adversarial accuracy 从 `83.59%` 小幅升到 `83.79%`，但 rank30 从 `84.18%`
降到 `82.81%`。全模型版本的未净化 AutoAttack 也只提高约 `0.48` 个百分点。
这说明当前收益并非简单来自“更新更多参数”；只更新 `block1/block2` 的 selective
RPCF 在参数效率和 rank30 净化性能上更好，为 purification-sensitive layer
selection 提供了初步消融证据。

去除 rank schedule、让六个 rank 始终均匀加权后，rank25 purified adversarial
accuracy 从 `83.59%` 升到 `83.98%`，但 rank30 从 `84.18%` 降到
`83.59%`，完整测试集 AutoAttack 也从 `72.26%` 降到 `71.55%`。因此动态
schedule 的作用更像是把适应重点从低 rank 逐步迁移到高 rank，而不是让所有
rank 同时提升。考虑当前 rank30 是最佳主结果，low-to-high schedule 仍应保留，
但论文表述需要明确这种 rank-dependent trade-off。

### EXP-019：五方法跨 seed 复验

EXP-019 在 `thubenchmark / EEGNet / seed43 / fold0 / eps0.03` 上重新运行完整
五方法流程：Madry AT、consistancy、RPCF selective、RPCF all-layers 和 RPCF
rank-weight uniform。该实验不复用 seed42 的最终指标，五种分类器分别接受自身
white-box AutoAttack，并在同一 seed43 n512 子集上执行 rank25/30 净化。

该实验用于检验 EXP-018 的两个主要消融判断是否跨 seed 稳定：

- selective layers 是否仍能以较少可训练参数保持或提升高-rank净化性能。
- low-to-high schedule 是否仍主要改善 rank30，而 uniform 权重偏向 rank25。

正式 seed43 结果已经完成。完整测试集未净化 AutoAttack 最好的是 Madry AT
（`79.52%`），rank25/30 净化后 adversarial accuracy 最好的是 consistancy
（`82.23%/82.62%`）。三个 RPCF 变体的完整 clean accuracy 均为 `93.45%`，
但完整 AutoAttack 只有 `71.31%` 至 `71.67%`；净化后 adversarial accuracy
也没有超过 consistancy。

这改变了 EXP-018 后的表述边界：seed42 上 RPCF selective 的 rank30 优势、
dynamic schedule 相对 uniform 的高-rank优势，在 seed43 上没有稳定复现。
因此当前不能把 RPCF selective + dynamic schedule 直接写成稳定默认方法。
更准确的判断是：consistancy 与 rank25/30 评估分布更匹配，而 RPCF 同时改变了
初始化方式、可训练层范围和六-rank训练分布；下一步如果继续推进 RPCF，需要把
“AT 初始化微调 vs 从头训练”、“敏感层选择 vs 全模型训练”、“六-rank schedule
vs rank25/30-only 增强”拆开做控制消融。

随后补跑了 EXP-019 的 consistancy six-rank版本：consistancy 也使用与 RPCF 一致的
rank `15,20,25,30,35,40` paired augmentation，再重新训练、攻击和 rank25/30
净化。结果显示 six-rank consistancy 的 full AutoAttack 为 `74.17%`，rank25/30
purified adversarial accuracy 为 `81.05%/79.49%`，低于 rank25/30-only
consistancy 的 `82.23%/82.62%`。在这个更公平的 six-rank训练分布口径下，
`RPCF rank-weight uniform` 的 rank25/30 purified adversarial accuracy
`82.03%/82.03%` 反而成为最好结果。

这说明此前 “consistancy 在 seed43 净化后最稳” 的判断需要进一步收窄：它更准确地
表示为 rank25/30-only augmentation 在 rank25/30 evaluation 上最匹配。若要求
consistancy 和 RPCF 使用同样的六-rank训练分布，则 consistancy 的优势不成立。
后续论文表述应把“训练 rank 分布是否匹配评估 rank”作为核心控制变量。

### EXP-024：RPCF_AT 跨 backbone 结果收窄了主张边界

EXP-024 将 RPCF_AT 从 EEGNet 扩展到 `tsception`、`conformer` 和 `atcnet`。结果显示，RPCF_AT 并不稳定支配 `Madry_AT + EEG_TNP`。`tsception` 是正例，rank25 净化后 adversarial accuracy 从 `48.63%` 提高到 `53.32%`；但 `conformer` 是明显负例，未净化 AutoAttack 从 `72.41%` 降到 `66.29%`，rank25/30 净化后也下降约 4 个百分点；`atcnet` 则在 Madry AT 已很强的情况下，RPCF_AT 全面小幅低于 Madry+TNP。

这意味着论文叙事不能再把 RPCF_AT 写成“跨模型稳定优于 Madry+TNP”的默认增强。更合适的表述是：RPCF_AT 是一种针对 EEG_TNP 净化分布的适配机制，它在 EEGNet 和部分 backbone/rank 上有效，但存在 backbone 依赖和层预算敏感性。后续 EXP-025 的意义正是检查固定 40% 层数规则是否过于粗糙，尤其是 atcnet 中 40% 逻辑层对应 66.44% 参数这一现象，可能导致过度微调或破坏原始 AT decision boundary。
